How the login authentication works:

  • 1. Your client will create a token that is sent to the server
  • 2. The server will recieve your client token and encrypt it
  • 3. The server will send a new encrypted token to your client
  • 4. Your client will send the login information through that encrypted token the server sent
  • 5. The server will check the login information in the token your client sent
  • 6. If everything checkout the server will send your client a new encrypted token telling it to login
  • 7. Your client will send an encrypted token to the server with the other encrypted token the server sent your client
  • 8. The server will then send your client an encrypted token with the user data allowing you to log in
